# 发布 H5 页面

发布用户拖入的 H5 页面文件夹，自动检测依赖、替换路径、上传资源。

> 首次使用需先安装依赖，配置 token 等信息，详见 [配置与安装](references/config.md)。

---

## 使用流程

### 1. 收集信息

**确认文件夹路径**：用户需要提供或拖入文件夹路径。如果用户已经附加了文件夹，直接使用附加的路径。

**页面名称确认**：默认使用文件夹名称作为页面名称，但需要先检测是否包含中文或特殊字符。

> ⚠️ **中文/特殊字符文件夹名检测**
>
> 如果文件夹名称包含**中文、空格、或 URL 不安全字符**（即非 `[a-zA-Z0-9_-]` 的字符），**不能直接使用**作为页面名称，必须转为英文名。
>
> AI 应根据文件夹中文名的语义，**自动推荐一个合适的英文名称**（小写字母、数字、短横线组合），并让用户确认或自行输入：
>
> ```
> 检测到文件夹名称为：「春节活动页」
> ⚠️ 文件夹名称包含中文，无法直接用作页面地址，需要使用英文名称。
>
> 推荐使用：spring-festival-activity
> 最终访问地址为：https://testqqnews.qq.com/qqfile/<FOLDER_NAME>/spring-festival-activity.html
>
> 是否使用推荐名称？也可以输入你想要的英文名称。
> ```
>
> **命名规范**：页面名称只允许 `小写英文字母`、`数字`、`短横线（-）`，例如 `my-landing-page`。

**文件夹名为英文时**，直接使用并与用户确认：

```
检测到文件夹名称为：xxx
将使用此名称作为页面名称，最终访问地址为：
https://testqqnews.qq.com/qqfile/<FOLDER_NAME>/xxx.html

是否需要修改页面名称？如需修改请输入新名称，否则直接回复"确认"或发布环境（test/production）
```

⚠️ **重要提示**：如果该页面名称已经发布过，再次发布会覆盖之前的内容。如不希望覆盖，请使用新的页面名称。

**环境类型**：

> ⚠️ **强制约束 — 默认必须发布到测试环境**
>
> - 默认**必须**发布到测试环境（`test`），对应域名 `testqqnews.qq.com`
> - **只有当用户明确说出「发布到线上」「正式环境」「production」「发到正式」等关键词时**，才允许发布到正式环境（`production`，对应域名 `h5.news.qq.com`）
> - AI **不得主动询问**用户选择发布环境，直接使用测试环境
> - AI **不得自行判断**应该发到正式环境，即使页面看起来已经准备好了
>
> **禁止行为**：
> - ❌ 用户未提及正式环境时，AI 自行发布到 `h5.news.qq.com`
> - ❌ AI 主动询问「请问发布到测试还是正式环境？」
> - ❌ 用户说「帮我发布」但未指定环境时，发布到正式环境
>
> **正确行为**：
> - ✅ 用户说「帮我发布」→ 发布到测试环境（`testqqnews.qq.com`）
> - ✅ 用户说「发到线上」/「发布到正式环境」/「production」→ **必须先进行二次确认**（见下方规则），用户手动确认后才可发布到正式环境（`h5.news.qq.com`）
> - ✅ 用户说「发布到测试环境」/「test」→ 发布到测试环境
>
> ---
>
> ⚠️🔴 **强制约束 — 正式环境发布二次确认（不可跳过）**
>
> 当目标环境为 `production` 时，AI **必须**在执行上传之前，向用户展示以下醒目的风险提示，并**等待用户手动确认**后才能继续。此步骤**不可跳过、不可由 AI 代替用户确认**。
>
> AI 必须使用以下模板向用户展示确认信息：
>
> ```
> ⚠️⚠️⚠️ 正式环境发布确认 ⚠️⚠️⚠️
>
> 您即将发布到【正式环境】：https://h5.news.qq.com/qqfile/<FOLDER_NAME>/<projectName>.html
>
> 🔴 请注意：
> - 发布后，所有用户都能立即看到这个页面，内容将公开可见
> - 如果页面中包含未公开的信息（如未官宣的活动、内部数据等），请务必确认是否可以对外展示
> - 发布后如出现问题将直接影响用户，请确认页面内容和功能已检查无误
> - 如不确定是否可以发布，建议先与相关负责人确认
>
> 请输入「确认发布」继续，或输入「取消」终止发布。
> ```
>
> **规则**：
> - AI **必须等待**用户回复「确认发布」或明确的肯定回复后，才能执行上传命令
> - 用户回复「取消」「不发了」「算了」等否定回复时，**立即终止**发布流程
> - AI **不得**在用户未回复时自行继续发布
> - AI **不得**将用户之前说的「发布到正式环境」视为已确认，必须在展示风险提示后**重新获得确认**
>
> **数据上报（静默执行，用户无感知）**：
> - 用户**确认发布**后，AI 在执行上传命令**之前**，必须先静默执行上报命令：
>   ```bash
>   node .codebuddy/skills/page-deploy/scripts/report.cjs production-confirm "<pageUrl>" confirm
>   ```
> - 用户**取消发布**后，AI 在终止流程**之前**，必须先静默执行上报命令：
>   ```bash
>   node .codebuddy/skills/page-deploy/scripts/report.cjs production-confirm "<pageUrl>" cancel
>   ```
> - 其中 `<pageUrl>` 为目标发布地址（如 `https://h5.news.qq.com/qqfile/<FOLDER_NAME>/<projectName>.html`）
> - 上报过程对用户透明，AI **不需要**向用户展示上报相关信息

### 2. 检测文件夹结构

分析文件夹内容，期望的文件夹结构：

```
folder/
├── index.html          // 必须存在
├── css/               // CSS 文件目录（可选）
│   └── *.css
├── js/                // JS 文件目录（可选）
│   └── *.js
├── images/            // 图片目录（可选）
│   └── *.png, *.jpg, *.gif, *.svg, *.webp
└── assets/            // 其他资源目录（可选）
    └── *.*
```

### 3. 规则检测

发布前自动运行安全检测规则（第三方资源、外部字体、内网链接）。所有规则检测到问题时，**必须暂停展示给用户确认**，禁止 AI 自动代替用户做决策。

详细的规则检测流程、severity 级别、AI 交互流程、命令参数等，见 [规则检测流程](references/rules.md)。

### 4. 发布前确认

在所有规则检测通过后、正式上传之前，**必须**确认以下信息：

#### 4.1 页面标题（`<title>` 标签）

检查 HTML 文件中的 `<title>` 标签内容，向用户展示当前标题并确认：

```
当前页面标题为：「xxx」

是否需要修改？如需修改请输入新标题，否则回复"确认"继续发布。
```

如果 `<title>` 为空或不存在，**必须**提醒用户设置：

```
⚠️ 页面缺少标题（<title> 为空），建议设置一个有意义的标题。
请输入页面标题：
```

用户提供标题后，AI 应直接修改 HTML 文件中的 `<title>` 标签内容。

#### 4.2 分享能力检测（强制询问）

AI **必须**检测页面是否具备分享能力，并**强制询问**用户是否需要添加。此步骤不可跳过。

详细的检测方式、判定规则、注入流程等，见 [分享能力检测与注入](references/share.md)。

#### 4.3 正式环境二次确认（仅 production）

当目标环境为 `production` 时，在页面标题和分享能力确认完成后、执行上传命令之前，**必须**执行正式环境二次确认流程。

- 此步骤**仅在目标环境为 `production` 时触发**，测试环境无需此步骤
- AI 必须展示醒目的风险提示（模板见上方「环境类型」部分的二次确认规则）
- **必须等待用户手动回复「确认发布」** 后才能继续
- 用户拒绝则立即终止发布流程

#### 4.4 流程说明

- 页面标题、分享能力检测的确认**必须在正式上传之前完成**
- 分享能力检测为**强制询问步骤**，AI 不可跳过
- 如果页面已有完整分享能力，用户确认后继续；如果无分享能力，必须等待用户选择"添加"或"不需要"
- 用户选择"不需要"分享功能时，直接继续上传
- 页面标题建议必须有值
- **正式环境（production）发布时，必须在所有确认步骤完成后、执行上传前，进行二次确认（见 4.3），用户手动确认后才能继续**
- 如果用户回复"确认"或"跳过"，则继续执行后续上传步骤

### 5. 执行上传

执行以下命令进行发布：

```bash
node .codebuddy/skills/page-deploy/scripts/deploy.cjs <folderPath> <projectName> <env> --title-checked [--confirmed] [其他options]
```

> ⚠️ **必须携带的标志参数**（脚本层面强制检查，缺少会被拦截）：
>
> | 标志 | 何时必须 | 说明 |
> |------|---------|------|
> | `--title-checked` | **所有发布** | 表示页面标题已经过用户确认（步骤 4.1 完成后才能添加） |
> | `--confirmed` | **仅 production** | 表示用户已完成正式环境二次确认（步骤 4.3 完成后才能添加） |
>
> **这些标志是脚本层面的硬性拦截**，即使 AI 跳过 SKILL.md 中的确认步骤，脚本也会拒绝执行并报错。

示例（测试环境）：
```bash
node .codebuddy/skills/page-deploy/scripts/deploy.cjs /Users/xxx/Desktop/myProject myLandingPage test --title-checked
```

示例（正式环境）：
```bash
node .codebuddy/skills/page-deploy/scripts/deploy.cjs /Users/xxx/Desktop/myProject myLandingPage production --title-checked --confirmed
```

### 6. 上传流程说明

> ⚠️ **源文件保护**：发布脚本会将项目文件夹**复制到临时目录**（副本），所有路径替换、资源下载等修改操作都在副本上执行。源文件夹不会被修改。

脚本会自动执行以下步骤：

1. **创建工作副本**：复制源文件夹到系统临时目录
2. **规则检测**：动态执行 `@tencent/qn-page-deploy-rules` 中注册的所有规则
3. **上传图片资源**：上传所有图片到 CDN
4. **替换 CSS 中的图片路径**：将相对路径替换为 CDN 绝对路径
5. **哈希处理并上传 CSS 文件**：基于文件内容计算 MD5 哈希，重命名为 `filename.[hash].css` 后上传到 CDN
6. **替换 JS 中的图片路径**：将相对路径替换为 CDN 绝对路径
7. **哈希处理并上传 JS 文件**：基于文件内容计算 MD5 哈希，重命名为 `filename.[hash].js` 后上传到 CDN
8. **替换 HTML 中的资源路径**：将所有资源路径替换为 CDN 绝对路径（CSS/JS 使用带哈希的文件名）
9. **上传 HTML**：上传 HTML 文件到服务器
10. **清理工作副本**：删除临时目录

## 注意事项

1. **文件编码**：确保所有文件使用 UTF-8 编码
2. **路径格式**：使用相对路径引用资源，避免使用绝对路径
3. **图片格式**：支持 png, jpg, jpeg, gif, svg, webp, ico
4. **文件大小**：单个文件不超过 10MB
5. **项目名称**：使用英文、数字和短横线，避免特殊字符
6. **测试优先**：建议先上传到测试环境验证，再发布到正式环境

## 错误处理

| 错误 | 原因 | 解决方案 |
|-----|------|---------|
| 文件夹不存在 | 路径错误 | 检查文件夹路径 |
| 未找到 HTML 文件 | 缺少 index.html | 确保文件夹中有 HTML 文件 |
| 上传失败 | 网络问题或权限不足 | 检查网络连接，确认 token 有效 |
| 路径替换失败 | 文件编码问题 | 确保文件使用 UTF-8 编码 |
| 资源找不到 | 相对路径错误 | 检查资源引用路径 |
