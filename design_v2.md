# 《巴菲特：从未亮出的那张底牌》V2 策划文档

> 本文档记录V2版本的核心逻辑变更，作为后续开发的参考基准。

---

## 一、核心变更概览

| 项目 | V1（旧版） | V2（新版） |
|------|-----------|-----------|
| 总卡牌数 | 24张（15成功+9失败） | **10张**（待定配比） |
| 用户面对的牌数 | 全部24张依次展示 | 随机抽取**6张** |
| 用户选择次数 | 6次投资机会（可在24张中选投） | 6张牌逐一选择（每张都必须选"投"或"过"） |

| 未选公司展示 | 有"查看其他牌面"区域 | **不展示**未被抽到的4张 |

## 二、新游戏流程

### 2.1 开屏页（保持不变）

- 主题标题、规则说明、"坐上牌桌"按钮
- 规则说明需更新为：
  - "巴菲特会给你看6张投资牌"
  - "每张牌，你必须选择：投资 或 放弃"
  - "选了就不能反悔"

### 2.2 发牌页

1. 从10张总牌库中**随机抽取6张**
2. 6张牌按抽取顺序（随机顺序）**逐一展示**
3. 每张牌用户**必须做出选择**（投资 IN / 放弃 PASS），不存在"跳过"的概念
4. 交互方式：左划PASS / 右划IN（与V1一致）
5. 状态栏显示：**第 X/6 张**（不再是X/24或X/25）
6. 6张全部选完后，进入结算页

**关键变化**：V1是"24张牌中选6次投资"，V2是"只给6张牌，每张都必须选"。

### 2.3 结算页（重大变更）

#### 总体布局

结算页由两部分组成：
1. **顶部**：头衔揭晓 + 统计数据
2. **主体**：6张纸牌横向/网格排列，初始显示**牌背**（带图腾图案）

#### 双翻牌机制

每张牌有**三个状态**，用户通过点击逐步揭晓：

| 状态 | 展示内容 | 视觉效果 |
|------|---------|---------|
| 初始（牌背） | 图腾图案 + 图腾名称 | 深墨绿底+金色描边，标准牌背设计 |
| 第一次翻面 | 公司名（谜底）+ 投资结局 + 用户选择对错标记 | 翻牌动画，正面展示公司信息 |
| 第二次翻面 | 巴菲特原话 + 投资理论 | 再次翻转动画，展示巴菲特的智慧 |

**交互流程**：
- 进入结算页 → 6张牌以牌背朝上排列
- 用户点击任意一张牌 → 触发翻牌动画 → 展示公司名+投资结局
- 用户再次点击同一张牌 → 再次翻牌动画 → 展示巴菲特点评
- 用户可以按任意顺序点击翻开6张牌
- 第二次翻面后，该卡牌不再可交互（已完全揭晓）

#### 未被抽到的4张牌

**不展示**。用户只看到自己实际面对过的6张牌的结果。

### 3.3 随机抽牌规则

- 6张抽出后，再随机打乱顺序依次展示

## 四、计分与头衔系统

### 4.1 计分规则

每张牌用户做出选择后，根据卡牌类型判定对错：

| 卡牌类型 | 用户选"投" | 用户选"过" |
|---------|-----------|-----------|
| 成功案例 | ✅ 正确（+1分） | ❌ 错误（+0分） |
| 失败案例 | ❌ 错误（+0分） | ✅ 正确（+1分） |

> 简化为"正确决策计数"（0-6分），不再使用复杂的收益率累加。

| 正确数 | 头衔 | 描述 |

| 5/6 | 🏆 价值投资大师 | 几乎完美！你深谙价值投资的精髓。 |
| 4/6 | 🏦 华尔街操盘手 | 出色的判断力！你在大多数关键时刻做出了正确的选择。 |

## 五、数据结构调整

### 5.1 卡牌数据字段

每张卡牌对象包含以下字段（保持与V1基本一致）：

```javascript
{
  id: Number,              // 卡牌编号 1-10
  totemName: String,       // 图腾名称（牌背展示）
  totemImg: String,        // 图腾图片路径
  riddle: String,          // 谜面（发牌页展示）
  answer: String,          // 谜底/公司名（第一次翻面展示）
  type: 'success' | 'failure',  // 案例类型（只保留两类）
  investResult: String,    // 投资结局（第一次翻面展示）
  theory: String,          // 投资理论（第二次翻面展示）
  buffettQuote: String,    // 巴菲特原话（第二次翻面展示）
  tipInvest: String,       // 选"投"后的即时提示语
  tipPass: String,         // 选"过"后的即时提示语
  isCorrectToInvest: Boolean  // 选"投"是否正确（成功案例=true，失败案例=false）
}
```

### 5.2 去掉的字段

- `scoreValue`：不再需要收益率分值，改用简单的对错计数
- `totemDesc`：图腾描述合并到图腾图片展示中

## 七、结算页卡牌翻转动画规格

### 7.1 翻转效果

- 使用CSS 3D transform实现Y轴翻转
- 翻转时长：0.6s，缓动函数：ease-in-out
- 翻转过程中卡牌有轻微缩放效果（scale从1→0.95→1）

### 7.2 三面内容

**面A - 牌背（初始状态）**
- 深墨绿底色
- 中央：图腾图片
- 底部：图腾名称
- 金色细描边

**面B - 第一次翻面（公司揭晓）**
- 背景色区分对错：
  - 用户选对 → 金色渐变背景
  - 用户选错 → 暗红色渐变背景
- 顶部：用户选择标记（✅投资正确 / ❌投资失误 / ✅放弃正确 / ❌放弃失误）
- 中央大字：公司名（谜底）
- 下方：投资结局摘要（2-3行）
- 底部提示："再次点击查看巴菲特怎么说"

**面C - 第二次翻面（巴菲特点评）**
- 统一的米白色/羊皮纸质感背景
- 顶部：巴菲特头像剪影图标
- 中央：巴菲特原话（加引号，斜体）
- 下方：投资理论精要（3-4行）
- 底部：该卡牌类型标签（成功投资/失败教训）

### 7.3 翻转逻辑

```
牌背(A) --[点击]--> 公司揭晓(B) --[点击]--> 巴菲特点评(C)
                                                    │
                                              [不可再交互]
```

## 九、重玩机制

- 点击"再来一局"后重新从10张中随机抽6张
- 由于是10选6，每次游戏体验不同（C(10,6)=210种组合）
- 增加了重玩价值和分享讨论空间
